Custom

JUI 라이브러리는 커스텀 컴포넌트를 쉽게 개발할 수 있도록 모듈 정의 메소드를 제공합니다.

Define

컴포넌트 모듈을 새로 정의하기 위해서는 해당 메소드의 매개변수를 생성할 모듈명과 로드할 모듈명 리스트를 설정하고, 마지막 콜백 함수를 통해 로드된 모듈 객체를 순서대로 받을 수 있다. 

jui.defineUI("ui.test", [ "util.base" ], function(_) {
	var UI = function() {
		this.init = function() {
			// 초기 구현부
		}
		
		this.func1 = function(val) {
			// 공개 메소드 1
		}
	}

    UI.setup = function() {
        return { // UI 옵션 기본값 설정
			option1: 1000,
			option2: true
        }
    }
	
	return UI;
}, "core");

UI.setup은 해당 컴포넌트에서 사용할 옵션을 정의하는 메소드이며, init은 컴포넌트 객체가 생성되었을 경우에 최초로 실행되는 메소드입니다. 그렇기 때문에 신규 컴포넌트를 정의하기 위해서는 필수적으로 구현해야 합니다.

Core

define 메소드의 리턴 클래스 내부에서는 this를 통해 UI 컴포넌트에 필요한 기본적인 메소드와 프로퍼티를 사용할 수 있습니다. 예를 들어 emit 메소드를 통해 커스텀 이벤트를 발생시킬 수 있고, 해당 컴포넌트를 사용하는 유저는 on 메소드나 컴포넌트 이벤트 옵션을 통해 커스텀 이벤트를 리스닝 할 수 있습니다.
